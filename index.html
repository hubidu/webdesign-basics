Der Elementbaum

Tags lassen sich übrigens auch schachteln (falls das noch nicht aufgefallen sein sollte). In diesem Fall beinhaltet das p Tag ein strong Tag. Block und Inline Elemente

HTML Tags fallen in eine von 2 Kategorien:
  • Block (bspw. p, h1-h6, ul) oder
  • Inline-Elemente (img, a, em, b)

Block Elemente

erstrecken sich immer über die gesamte Seitenbreite und werden automatisch aufeinandergestapelt (wie Umzugskartons). Sie werden nur so hoch wie ihr Inhalt. Ein häufig benutztes Block Element ist div (was soviel heisst wie "mach ein Kästchen").

Organization der Seite

Jede halbwegs größere Website besteht aus verschiedenen Teilen. Lasst uns nun mit den folgenden Tags etwas Struktur in unsere Seite bringen.
  • Einen header mit unserem Branding und der Navigation

Cascading Style Sheets

Mittlerweile sieht unsere Seite ziemlich hässlich aus. Bringen wir also ein bisschen Style, d. h. Farbe, Schriftarten, Abstände und Positionierung rein. Dazu verwenden wir CSS. CSS ist ein wenig wie Malen mit Code

CSS ist wie HTML eine Sprache, verwendet aber eine komplett andere Syntax. Um CSS Code in einer HTML Datei unterzubringen verwenden wir das style Tag. Innerhalb dieses Tags schreiben wir dann CSS.

CSS existiert, weil wir den Inhalt unserer Seite vom Aussehen trennen wollen. Was damit alles möglich ist zeigt CSS Zen Garden

Textfarbe ändern

Lasst uns zunächst eine sehr subtile Änderung vornehmen und den Kontrast des Textes etwas reduzieren. 

      body {
        color: #555;
      }

      h1,
      h2,
      h3,
      strong {
        color: #333;
      }

CSS Syntax: wer { was: wie; }

Oder auch selector { property: value }

  • der selector wählt das Ziel HTML Element aus (in obigem Beispiel body)
  • property definiert welche Eigenschaft geändert werden soll (im Beispiel der Rand und die Breite des Elements)
  • value definiert wie die Eigenschaft geändert wird bzw. welchen Wert sie erhält

Im obigen Beispiel ist der selector das body element, das property ist color (die Schriftfarbe) und der value ist #555 (ein hexadecimal codierter Farbwert im RGB format).

Außerdem seht ihr im obigen Beispiel, dass ein selector auch aus mehreren Tags bestehen kann. Diese werden dann durch Komma voneinander getrennt. Damit lassen sich mehrere Tags auf einmal referenzieren.

Als Farbwert könnt ihr auch vordefinierte Werte wie blue, green, gray etc. verwenden. Probiert das doch gleich mal aus!

Seite zentrieren

Da lange Textzeilen schwert zu lesen sind, wollen wir als nächstes die Seitenbreite begrenzen und den Inhalt zentrieren. Dazu geben wir dem body Tag eine feste Breite (em ist eine Längeneinheit, die sich mit der Schriftgröße anpasst). Außerdem setzen wir den linken und rechten Rand des Body Elements auf auto. Damit zentriert sich die Seite automatisch. 

      body {
        margin: 0 auto;
        width: 50em;
      }

CSS Box Model

Wie anfangs erwähnt sind HTML Elemente eigentlich nur kleine Kästchen (engl. Boxes). Die Größe der Rechtecke ist dabei dynamisch, d. h. sie hängt vom Inhalt ab. Der Inhalt drückt also sozusagen die Box auf.

Die Größe der Box richtet sich aber nicht nur nach dem Inhalt, sondern hängt auch ab von:

  • Padding
  • Border
  • Margin
Siehe dazu auch CSS Box Model.

Font und Abstände

Lasst uns jetzt eine weitere kleine Änderung machen. Die Standardschriftart des Browsers (Times) sieht nicht besonders schön aus und auch die Zeilenabstände tragen nicht zur Lesbarkeit bei. 

      body {
        font-family: "Helvetica", "Arial", sans-serif;
      }

      body {
        line-height: 1.5;
        padding: 4em 1em;
      }

      h2 {
        margin-top: 1em;
        padding-top: 1em;
      }

Sieht gleich viel besser aus, oder?

Balance

Lasst uns gleich auch noch die Code Abschnitte verschönern.

      code,
      pre {
        background: #eee;
      }

      code {
        padding: 2px 4px;
        vertical-align: text-bottom;
      }

      pre {
        padding: 1em;
      }

Custom Font

Und weil Text so wichtig ist, verwenden wir gleich noch einen sog. Custom Font (Roboto) , weil der einfach schöner aussieht. 

      @import 'https://fonts.googleapis.com/css?family=Roboto:300,400,500';

      body {
        font-family: "Roboto", "Helvetica", "Arial", sans-serif;
      }

Sieht noch ein bisschen klobig aus. Deswegen wählen wir noch eine schlankere Variante (siehe auch font-weight) für die Überschriften. 

      h1,
      h2,
      h3 {
        font-weight: 500;
      }

CSS Refactoring

Refactoring is a disciplined technique for restructuring an existing body of code, altering its internal structure without changing its external behavior.

Häh? Auf Deutsch: Unsere HTML Datei ist mittlerweile schon recht groß geworden. Lasst uns deshalb den HTML und CSS Code voneinander trennen (ist ja auch die eigenliche Absicht von CSS) und das CSS in eine eigene Datei packen.

Legt dazu jetzt bitte eine neue Datei styles.css an und kopiert den CSS Code hinein. Dann referenzieren wir die CSS Datei mittels folgendem link Tag: 

      <link rel="stylesheet" type="text/css" href="styles.css" />

Hintergrundbild im Header

Bilder und Icons dienen entweder der optischen Verschönerung der Seite oder sie unterstützen aktiv den Content.

Sucht euch auf Unsplash ein schönes Photo aus und dann bauen wir es als Hintergrundbild in den Header ein. 

      body {
        padding: 0;
        margin: 0;
        max-width: none;
      }
      main {
        margin: 0 auto;
        max-width: 50em;
      }
      header {
        background-color: #263d36;
        background-image: url("https://hd.unsplash.com/photo-1470214304380-aadaedcfff84");
        background-size: cover;
        line-height: 1.2;
        padding: 8vw 2em;
        text-align: center;
      }
Experimentiert mit verschiedenen Bildern, indem ihr andere Werte in background-image: url(...) verwendet.

Ändert außerdem bitte den HTML Code des Headers wie folgt:

      <header>
        <p>
          <h1>Wir bauen eine Website!</h1>
        </p>
        <a href="home.html">Home</a>
        <a href="about.html">Über</a>
        <a href="contact.html">Kontakt</a>
      </header>

Und so funktioniert's

  1. Zunächst sorgen wir dafür, dass sich das Bild über die gesamte Seitenbreite erstrecken kann, indem wir die automatische Zentrierung des body Elements aufheben und sie stattdessen auf das main Element anwenden.
  2. Dann fügen wir das Bild per CSS background-image als Hintergrundbild des header Elements ein.
  3. Da das Bild größer als der Header ist, kontrollieren wir mittels background-position welcher Teil des Bildes im sichtbaren Bereich liegt. Wir verwenden center weil wir es horizontal zentrieren wollen. Durch Angabe von top zeigen wir das Bild beginnend mit seinem oberem Rand.
  4. background-size: cover zieht das Bild dabei auf die gesamte Breite des header Elements.
  5. Mit text-align: center sorgen wir dafür, dass der Text im header mittig ausgerichtet wird.

Gestaltung des Header

Je nachdem, ob ihr ein eher helles oder eher dunkles Bild gewählt habt, ist die Überschrift jetzt mehr oder weniger gut lesbar. Lasst uns deshalb die Farbe und die Schriftgröße anpassen. 

      header h1 {
        margin: 0;
        margin-bottom: 1em;
        padding: 0;
        color: white;
        font-size: 5em;
        font-weight: 300;
        text-shadow: 2px 4px 3px rgba(0,0,0,0.3);
      }

Und so funktioniert's

  1. Zunächst bemerkt ihr einen neuen CSS Selector header h1. Es handelt sich um einen sog. descendant selector. Er wirkt gezielt auf das h1 Element im header Element und wir können dessen Eigenschaften überschreiben.
  2. Mit margin und padding reduzieren wir den Leerraum um unsere Überschrift.
  3. Mit font-size machen wir die Überschrift größer als eine Standard h1 Überschrift
  4. Und mit text-shadow erzeugen wir noch einen netten Schatten unter dem Text, der die Lesbarkeit verbessert.
