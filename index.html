Der Elementbaum

Tags lassen sich übrigens auch schachteln (falls das noch nicht aufgefallen sein sollte). In diesem Fall beinhaltet das p Tag ein strong Tag. Aufpassen muss man nur, dass Kindelemente (in diesem Fall strong), immer abgeschlossen sein müssen, bevor das Elternelement (in diesem Fall p) geschlossen wird.

Durch die Schachtelung entsteht eine baumartige Struktur (auch Elementtree genannt)

Block und Inline Elemente

HTML Tags fallen in eine von 2 Kategorien:
  • Block (bspw. p, h1-h6, ul) oder
  • Inline-Elemente (img, a, em, b)

Block Elemente

erstrecken sich immer über die gesamte Seitenbreite und werden automatisch aufeinandergestapelt (wie Umzugskartons). Sie werden nur so hoch wie ihr Inhalt. Ein häufig benutztes Block Element ist div (was soviel heisst wie "mach ein Kästchen").

Organization der Seite

Jede halbwegs größere Website besteht aus verschiedenen Teilen. Lasst uns nun mit den folgenden Tags etwas Struktur in unsere Seite bringen.
  • Einen header mit unserem Branding und der Navigation
  • Einen main content
  • Einen sidebar mit dem Inhaltsverzeichnis
  • Einen footer für das Impressum und zusätzlichen, weniger wichtigen Links

Cascading Style Sheets

Mittlerweile sieht unsere Seite ziemlich hässlich aus. Bringen wir also ein bisschen Style, d. h. Farbe, Schriftarten, Abstände und Positionierung rein. Dazu verwenden wir CSS. CSS ist ein wenig wie Malen mit Code

CSS ist wie HTML eine Sprache, verwendet aber eine komplett andere Syntax. Um CSS Code in einer HTML Datei unterzubringen verwenden wir das style Tag. Innerhalb dieses Tags schreiben wir dann CSS.

CSS existiert, weil wir den Inhalt unserer Seite vom Aussehen trennen wollen. Was damit alles möglich ist zeigt CSS Zen Garden

Textfarbe ändern

Lasst uns zunächst eine sehr subtile Änderung vornehmen und den Kontrast des Textes etwas reduzieren. 

      body {
        color: #555;
      }

      h1,
      h2,
      h3,
      strong {
        color: #333;
      }

CSS Syntax: wer { was: wie; }

Oder auch selector { property: value }

  • der selector wählt das Ziel HTML Element aus (in obigem Beispiel body)
  • property definiert welche Eigenschaft geändert werden soll (im Beispiel der Rand und die Breite des Elements)
  • value definiert wie die Eigenschaft geändert wird bzw. welchen Wert sie erhält

Im obigen Beispiel ist der selector das body element, das property ist color (die Schriftfarbe) und der value ist #555 (ein hexadecimal codierter Farbwert im RGB format).

Außerdem seht ihr im obigen Beispiel, dass ein selector auch aus mehreren Tags bestehen kann. Diese werden dann durch Komma voneinander getrennt. Damit lassen sich mehrere Tags auf einmal referenzieren.

Als Farbwert könnt ihr auch vordefinierte Werte wie blue, green, gray etc. verwenden. Probiert das doch gleich mal aus!

Seite zentrieren

Da lange Textzeilen schwert zu lesen sind, wollen wir als nächstes die Seitenbreite begrenzen und den Inhalt zentrieren. Dazu geben wir dem body Tag eine feste Breite (em ist eine Längeneinheit, die sich mit der Schriftgröße anpasst). Außerdem setzen wir den linken und rechten Rand des Body Elements auf auto. Damit zentriert sich die Seite automatisch. 

      body {
        margin: 0 auto;
        width: 50em;
      }
